conso — Features & Parity

← Planning hub · what we build, measured against the paid app

conso aims for full feature parity with the paid Mole for Mac app (mole.fit) — clean, software, optimize, analyze and monitor — surpasses it in a few honest places, and is straight about the one capability no third-party app can deliver on Apple Silicon: fan control. Everything below is scoped to what a clean-room, non-sandboxed SwiftUI app on macOS 26 / M4 Max can actually do, using only standard OS tools.

Where conso goes further

  • Three switchable themes + light/dark. A semantic ThemeStore drives the whole UI — Modern Native (Liquid Glass), Pro Dark (Vercel/Geist mono), and Character (liquid-glass + the conso mascot), each honoring system light/dark. Themes persist across the app.
  • Own clean-room brand. All logic written fresh against standard OS tools; no Mole code, no GPL obligations, and deliberately none of Mole's globe / solar-system / "Worlds" motif. conso's identity is its own.
  • Doctor — live permission verification & fail-loud. Rather than trusting a cached "granted," Doctor probes the real state every launch and pre-flights before every operation: Full Disk Access (by reading a known protected path, not the toggle), the privileged helper (XPC ping + SMAppService status), code signing, disk space and update health. Anything missing is surfaced with a button straight to the exact Settings pane — and a clean that would silently skip protected files is stopped, not degraded.
  • Top-centered glass nav. A single top-centered glass pill nav unifies all five pillars (Clean · Software · Optimize · Analyze · Status), with Doctor and Settings in the trailing corner — a calmer, more native layout than a left sidebar.
  • First-class developer junk. DerivedData, node_modules and package caches are their own clean category — built for an M4 Max dev machine.

Accessibility

Accessibility is a first-class commitment, not an afterthought — conso honors every relevant system setting:

  • Reduce Transparency — the Liquid-Glass blur is replaced with a fully opaque fallback so text stays legible.
  • Reduce Motion — animated transitions and live sparklines are calmed or cross-faded instead of moved.
  • Increase Contrast — hairlines, borders and tag colors strengthen to meet contrast needs.
  • Dynamic Type — the UI scales with the system text size rather than locking to fixed points.
  • VoiceOver labels — every control, stat and row carries a descriptive label and value, so the app is fully narratable.

Honest limits

We do not over-promise. The items below are real constraints of the platform and our distribution choice — stated up front, not buried.
  • Fan control is impossible on Apple Silicon. Apple exposes no third-party fan-control interface on M-series Macs — the SMC fan APIs that worked on Intel are gone for apps. conso therefore reads fan RPM and thermals only and never claims to set them. The paid Mole app's "fan control" is effectively Intel-Mac only; on Apple Silicon nobody can do it.
  • Some features need Full Disk Access + a privileged helper. Scanning/cleaning protected folders and building a complete treemap require Full Disk Access. Root-only work — deep cache purge under system paths, certain maintenance routines, and detailed powermetrics sensor reads — runs through a small audited XPC helper installed once via SMAppService (one admin prompt). Without these grants, the affected features are blocked and clearly flagged by Doctor, never silently skipped.
  • Personal-first signing. conso ships self-signed at $0, so first launch needs the Gatekeeper override: System Settings → Privacy & Security → Open Anyway (the right-click → Open shortcut was removed in macOS 15). No Apple Developer account means no notarization and no Mac App Store until/unless we move to Developer ID ($99/yr) + Sparkle for sharing.
